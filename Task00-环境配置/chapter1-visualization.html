▶ 1.1 什么是智能体？

在人工智能领域，智能体被定义为任何能够通过传感器（Sensors）感知其所处环境（Environment），并自主地通过执行器（Actuators）采取行动（Action）以达成特定目标的实体。

四个基本要素：

  • 环境：智能体所处的外部世界
  • 传感器：持续感知环境状态（摄像头、麦克风、API等）
  • 执行器：改变环境状态的工具（机械臂、代码执行等）
  • 自主性：基于感知和内部状态进行独立决策的能力
▶ 1.1.2 大语言模型驱动的新范式

以 GPT 为代表的大语言模型正在显著改变智能体的构建方法。LLM 智能体与传统智能体的核心区别：

  • 核心引擎：从显式编程规则 → 隐式学习的神经网络
  • 知识来源：从人工构建知识库 → 海量数据预训练
  • 交互方式：从结构化指令 → 自然语言对话

三大核心能力：

  1. 规划与推理：将高层级目标分解为逻辑子任务
  2. 工具使用：主动调用外部工具补全信息缺口
  3. 动态修正：根据反馈调整后续行动
▶ 1.2.2 智能体的运行机制

智能体循环（Agent Loop）是智能体与环境交互的核心机制：

  1. 感知（Perception）：通过传感器接收环境输入
  2. 思考（Thought）：
    • 规划（Planning）：制定或调整行动计划
    • 工具选择（Tool Selection）：选择合适的工具和参数
  3. 行动（Action）：通过执行器执行具体操作
  4. 观察（Observation）：环境状态变化反馈

这个循环持续重复，直到达成目标。

▶ 1.2.3 智能体的感知与行动

Thought-Action-Observation 交互协议是 LLM 智能体的标准交互格式：

Thought: 用户想知道北京的天气。我需要调用天气查询工具。
Action: get_weather("北京")
Observation: 北京当前天气为晴，气温25摄氏度，微风。
  • Thought：智能体内部决策的"快照"，自然语言形式
  • Action：具体操作，通常是函数调用
  • Observation：环境反馈，处理成自然语言文本
▶ 1.3 动手体验：5分钟实现第一个智能体

目标：构建一个智能旅行助手，能查询天气并推荐景点。

准备工作：

pip install requests tavily-python openai

核心步骤：

  1. 指令模板：设计 system_prompt 告诉 LLM 角色、工具和输出格式
  2. 工具定义：
    • get_weather(city)：查询真实天气
    • get_attraction(city, weather)：搜索推荐景点
  3. 接入 LLM：使用 OpenAI 兼容接口
  4. 执行循环：构建 Prompt → 调用 LLM → 解析 Action → 执行工具 → 记录 Observation

P1 进阶理解 理解智能体生态和选型的背景知识
▶ 1.1.1 传统视角下的智能体

传统智能体的演进路径：

  • 反射智能体（Simple Reflex Agent）：基于"条件-动作"规则，如恒温器
  • 基于模型的反射智能体（Model-Based）：拥有内部世界模型，能追踪无法直接感知的状态
  • 基于目标的智能体（Goal-Based）：主动规划达成特定目标，如 GPS 导航
  • 基于效用的智能体（Utility-Based）：在多个目标间权衡，最大化期望效用
  • 学习型智能体（Learning Agent）：通过强化学习自我改进，如 AlphaGo
▶ 1.1.3 智能体的类型

基于决策时间维度：

  • 反应式智能体：即时响应，速度快但短视（如安全气囊）
  • 规划式智能体：深思熟虑，战略性强但耗时（如棋手）
  • 混合式智能体：结合两者优点，分层设计或循环模式

基于知识表示：

  • 符号主义 AI：显式规则和逻辑，可解释但脆弱
  • 亚符号主义 AI：神经网络，强大但黑箱
  • 神经符号主义 AI：融合两者，类似卡尼曼的双系统理论
▶ 1.2.1 任务环境定义

PEAS 模型用于精确描述任务环境：

  • Performance：性能度量（如用户满意度）
  • Environment：环境（如互联网、数据库）
  • Actuators：执行器（如 API 调用、语音输出）
  • Sensors：传感器（如用户输入、API 返回）

环境特性：

  • 部分可观察、随机性、多智能体、序贯、动态
▶ 1.4 智能体应用的协作模式

1.4.1 作为开发者工具：

  • GitHub Copilot：代码自动补全
  • Claude Code：终端中的 AI 编程助手
  • Cursor：AI 原生代码编辑器

1.4.2 作为自主协作者：

  • 单智能体自主循环（AgentGPT）
  • 多智能体协作（CAMEL、MetaGPT、CrewAI）
  • 高级控制流架构（LangGraph）

1.4.3 Workflow vs Agent：

  • Workflow：预定义流程，确定性强，适合标准化任务
  • Agent：目标导向，自主决策，适合复杂动态任务
