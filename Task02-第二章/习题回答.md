1、对于第一问：
充分性论断： 只要有一个物理符号系统，就足以产生通用智能。
必要性论断： 任何展现智能的系统，本质上一定是个物理符号系统。
为我觉得是在实践挑战上：符号主义在处理非结构化常识、应对动态环境的“框架问题”以及处理知识获取的规模化时遇到了瓶颈。
LLM的符合性： LLM在底层是联结主义（数值计算），但在应用层（如工具调用、逻辑推理）表现出极强的符号处理特征。它更像是一个由联结主义实现的物理符号系统 
2、对于第二问：
虽然专家系统取得了显著成功，但是在实际医学研究中，很多个体的出现的情况是不一样的，对于医疗对象的人来说，如果出现错误回答，将是灾难性的，还有就是最终责任归属人的问题，如医生对黑盒/固定规则的不信任感
但是最终还是要在这种不信任之下为它负责，这显然是不对的。
现代设计方案：这让为我想到了“蚂蚁阿福”这种医疗智能体，或许采用“人机协作”模式。利用 LLM 处理非结构化病历，结合 RLHF（人工反馈强化学习）对齐医学伦理，并引入可解释的知识图谱进行双重验证，克服纯规则的脆弱性现代设计方案： 采用“人机协作”模式。利用 LLM 处理非结构化病历，结合 RLHF（人工反馈强化学习）对齐医学伦理，并引入可解释的知识图谱进行双重验证，克服纯规则的脆弱性
法律条文判定、财务审计、自动化工厂的安全停机逻辑。这些领域要求100%的可解释性和确定性不允许概率性错误。
3、对于第三问：
本质差异（对比 ChatGPT）：语义： ELIZA 是模式匹配，不理解词义；ChatGPT 拥有语义向量空间。状态： ELIZA 无状态；ChatGPT 有长短期记忆和上下文窗口。泛化： ELIZA 只能回答预设词；ChatGPT 拥有跨领域的常识涌现能力。
如果要用规则覆盖所有对话，输入N个词，可能的组合是指数级的。数学表达：对于长度为L的句子，若词表大小为V，可能的组合数为V^L。手工规则无法穷举这种搜索空间。
4、对于第四问：
GRASP失效：整个“搭建”任务会中断，因为底层依赖断裂。
优势上：其模块化强，易于扩展新功能。
劣势上：缺乏全局协调可能导致局部冲突，调试复杂。
与现代系统关联：MetaGPT 或 CrewAI 正是明斯基思想的工程化，将任务分配给不同角色的“小智能体”（Agent），通过通信协议达成涌现。
对于是否过时，我觉得并未过时。 LLM 虽强，但在处理极其复杂的长链任务时，仍需将其作为“大脑”，配合记忆和工具模块（即不同的 Agency）来工作 
5、对于第五问：
AlphaGo 并不被告知哪步棋是“对”的，而是通过成千上万局对弈的胜负（奖励信号）不断微调策略概率。
监督学习需要“人类标注的正确答案”；RL 需要“环境反馈的评分”。
马里奥任务： 强化学习更合适。因为游戏是序贯决策，很难有完美的人类数据集覆盖所有状态，而 RL 可以通过不断的死亡和复活找到最优路径。
LLM中的作用： RLHF（基于人类反馈的强化学习）。它让 LLM 的输出更符合人类的偏好和价值观。
6、对于第六问：
预训练通过“海量文本预测下一个词”实现了自监督学习，不再需要人工编写规则 
本质区别：符号主义是显式、硬编码的；预训练是隐式、参数化的压缩表示。
互联网数据问题：包含偏见、虚假信息和低质量内容。缓解方案：数据清洗、对齐（Alignment）技术。
未来展望：可能会向“持续学习”或“实时更新”范式演进，解决预训练模型知识过时的问题
7、对于第七问:
符号主义时代： 编写大量静态语法检查规则（Linting）。可能会无法理解逻辑意图，只能查错别字。
2015深度学习时代： 使用 RNN/LSTM 做序列标注。但是无法处理长距离逻辑，难以给出改进建议。
现代 Agent 架构包含的模块：
感知： 接入 Git API 获取 Diff 内容。
思考（LLM + 规划）： LLM 分析代码意图，拆解审查任务（逻辑、安全、性能）。
记忆： 检索该项目的历史代码规范和以往的 Bug 记录。
行动： 调用静态分析工具，最终生成总结报告。
